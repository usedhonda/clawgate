# ClawGate Character Specification v1

Create your own desktop pet character for ClawGate.

## Quick Start

1. Create a folder: `~/.clawgate/characters/your-character/`
2. Add a `manifest.json` (see below)
3. Add your sprite images (PNG, transparent background)
4. Add a `preview.png` (thumbnail for character selection)
5. Restart ClawGate — your character appears in the selection menu

## Folder Structure

```
~/.clawgate/characters/your-character/
  manifest.json     # Required: character definition
  preview.png       # Required: thumbnail (any size, displayed at 64x64)
  idle-01.png       # Individual frame files...
  idle-02.png
  walk-sheet.png    # ...or sprite sheet files
```

## manifest.json

```json
{
  "name": "your-character",
  "displayName": "My Character",
  "author": "Your Name",
  "version": "1.0.0",
  "description": "A brief description of your character",
  "states": [
    {
      "name": "idle",
      "frames": ["idle-01.png", "idle-02.png", "idle-03.png"],
      "fps": 3,
      "loop": true
    }
  ]
}
```

### Top-Level Fields

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Unique identifier (lowercase, no spaces, used internally) |
| `displayName` | No | Human-readable name shown in UI |
| `author` | No | Creator name |
| `version` | No | Semver version string |
| `description` | No | Short description |
| `states` | Yes | Array of animation states |

### State Object

Each state defines one animation (idle, walk, speak, etc.):

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | State identifier (see Standard States below) |
| `frames` | Yes | Array of PNG filenames (individual frames OR a single sprite sheet) |
| `fps` | No | Frames per second (default: 4) |
| `loop` | No | Loop animation (default: true) |
| `sheetColumns` | No | If frames[0] is a sprite sheet: number of columns |
| `sheetRows` | No | If frames[0] is a sprite sheet: number of rows |

### Two Frame Modes

**Mode 1: Individual frames** — list each frame file:
```json
{
  "name": "walk-right",
  "frames": ["walk-r-01.png", "walk-r-02.png", "walk-r-03.png", "walk-r-04.png"],
  "fps": 6,
  "loop": true
}
```

**Mode 2: Sprite sheet** — one file with grid layout:
```json
{
  "name": "react",
  "frames": ["react-sheet.png"],
  "sheetColumns": 3,
  "sheetRows": 2,
  "fps": 2,
  "loop": false
}
```
Frames are read left-to-right, top-to-bottom.

## Standard States

Your character should support at least `idle`. Other states are optional — ClawGate falls back to `idle` if a state is missing.

### Required

| State | Description | Recommended Frames |
|-------|-------------|-------------------|
| `idle` | Default standing pose | 2-8 frames, loop |

### Recommended

| State | Description | Recommended Frames |
|-------|-------------|-------------------|
| `speak` | Talking (mouth movement) | 4-8 frames, loop |
| `blink` | Eye blink cycle | 4-6 frames, no loop |
| `walk-right` | Walking right (side view) | 4-8 frames, loop |
| `walk-left` | Walking left (side view) | 4-8 frames, loop (or flip walk-right) |
| `walk-front` | Walking toward viewer | 4-8 frames, loop |
| `walk-back` | Walking away from viewer | 4-8 frames, loop |

### Optional

| State | Description | When Used |
|-------|-------------|-----------|
| `idle-breathe` | Subtle idle movement (sway, breathe) | Random idle variation |
| `speak-mix` | Transition from idle to speak | Message arrival |
| `speak-tilt` | Speaking with head tilt/gestures | Long messages |
| `talk` | Alternative speaking animation | Random variation |
| `wave` | Waving hand | Greeting / notification |
| `react` | Reaction expressions (surprise, happy, etc.) | Events from Gateway |
| `blush` | Blushing/embarrassed | Special events |

You can also define **custom states** — they won't be auto-triggered but can be used via the API.

## Image Requirements

- **Format**: PNG with transparent background (or solid green #00FF00 for chroma key)
- **Size**: Any size. Displayed at 128x128 by default (configurable)
- **Consistency**: All frames for one character should have the same proportions
- **Orientation**: Character should face right for `walk-right`, left for `walk-left`

### Tips

- Keep character centered in each frame
- Use the same canvas size for all frames in a state
- For walk cycles: 4 frames is enough (see Shimeji pattern)
- `walk-left` can be auto-generated by flipping `walk-right` — just provide one

## Creating Sprites with AI

You can use Gemini (Nano Banana 2) to generate sprites:

1. Create a reference image of your character (single pose)
2. Use Gemini 2.5 Flash Image API or gemini.google.com web UI
3. Attach reference image + prompt describing the animation
4. Use green #00FF00 background for easy removal

Example prompt:
```
Using the attached character as EXACT reference.
Generate a 1x4 horizontal strip on bright green #00FF00 background.
Walk cycle facing RIGHT, 4 frames:
Frame 1: Right foot forward
Frame 2: Mid-stride
Frame 3: Left foot forward
Frame 4: Mid-stride returning
BACKGROUND: solid bright green #00FF00 (NOT white)
```

### Consistency Tips
- Always attach a reference image
- Explicitly state what should NOT change ("same outfit, same hair")
- Explicitly exclude unwanted elements ("NO ribbon, NO bow tie")
- Generate all animations in the same Gemini chat session
- Specify green background 3 times in the prompt (once isn't enough)

## Example Characters

### Minimal (idle only)
```
my-char/
  manifest.json
  preview.png
  idle.png        # single static image
```
```json
{
  "name": "my-char",
  "states": [
    { "name": "idle", "frames": ["idle.png"], "fps": 1 }
  ]
}
```

### Full Featured
```
my-char/
  manifest.json
  preview.png
  idle-sheet.png
  blink-sheet.png
  speak-sheet.png
  walk-right-sheet.png
  walk-left-sheet.png
  react-sheet.png
```
